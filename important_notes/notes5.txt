Kubernetes me kya-kya namespace ke andar hota hai?

    Ye sab namespace ke andar hota hai:
    Pods
    Services
    Deployments
    ConfigMaps
    Secrets
    Ingress
    Jobs
    Ye sab namespace scoped hote hain.


    But kuch cheeze cluster-wide hoti hain:
    Nodes
    PersistentVolumes
    StorageClass
    Namespace itself
    Ye cluster scoped hoti hain.




To get all resource we will use: 
    kubectl get all --namespace=kube-system




MULTI CONATINER POD:

- let's say a pod running on kubernets cluster and it has one app container eg. nginx.   
  Now we can have additonal containers to support this app container and this container could be an init container that runs before the app container to do certain tasks or this could be sidecar/helper container that runs all the time with app conatiner and it provides certain input or it takes certain output from the app container 



- When you run kubectl create and the YAML file doesn't specify a namespace in the metadata section, kubectl looks at your active configuration (kubeconfig) to decide where to put the resource.

- we can verifty current context : 
    kubectl config view --minify --output 'jsonpath={..namespace}'
- switch back to default context : 
    kubectl config set-context --current --namespace=default
